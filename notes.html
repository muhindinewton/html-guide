Chapter 1: Introduction to HTML

üí° What is HTML?

HTML stands for HyperText Markup Language.

  • HyperText means text with links to other documents
  • Markup means we "mark up" text to give it structure
  • Language means it has rules and syntax

Important Things to Know:

  • ‚úÖ HTML is NOT a programming language‚Äîit's a markup language
  • ‚úÖ It tells your web browser what to display and how to organize content
  • ‚úÖ We are using HTML5, the latest version
  • ‚úÖ Every website you visit uses HTML!
üåç Real-World Example:

Think of HTML like the skeleton of a building:

  • The skeleton (HTML) gives structure
  • Paint and decorations (CSS) make it look beautiful
  • Electricity and plumbing (JavaScript) make it functional

Chapter 2: Your First Webpage Structure

üèóÔ∏è The Basic HTML Skeleton

Every HTML page MUST have this structure. Memorize it!

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Webpage</title>
</head>
<body>
    <h1>Hello World!</h1>
    <p>This is my first webpage.</p>
</body>
</html>

üìù Understanding Each Part:

Tag Purpose
<!DOCTYPE html> Tells the browser "This is HTML5"
<html lang="en"> Root element. lang="en" means English language
<head> Contains information ABOUT the page (not visible)
<meta charset="utf-8"> Supports special characters like √°, √±, ‚Ç¨
<title> Shows in browser tab
<body> Everything users SEE goes here
üí° Pro Tip: Always start EVERY HTML file with this skeleton!

Chapter 3: Working with Text

3.1 Headings

HTML has 6 levels of headings, from biggest (h1) to smallest (h6).

<h1>Main Title (Use only ONCE per page)</h1>
<h2>Section Title</h2>
<h3>Sub-section</h3>
<h4>Smaller heading</h4>
<h5>Even smaller</h5>
<h6>Tiniest heading</h6>
‚ö†Ô∏è Important: Use only ONE h1 per page!

Chapter 4: Creating Lists

4.1 Unordered Lists (Bullets)

<ul>
    <li>Milk</li>
    <li>Bread</li>
    <li>Butter</li>
</ul>

4.2 Ordered Lists (Numbers)

<ol>
    <li>Mix flour</li>
    <li>Add eggs</li>
    <li>Bake</li>
</ol>

4.3 List Attributes

<ol start="5" type="A">
    <li>Math</li>
    <li>English</li>
</ol>

Types: "1" (numbers), "A" (uppercase letters), "a" (lowercase), "I" (Roman numerals)

4.4 Nested Lists

<ul>
    <li>Subjects
        <ol>
            <li>Math</li>
            <li>Science</li>
        </ol>
    </li>
</ul>

Chapter 5: Links and Images

5.1 Links (Anchor Tag)

<a href="https://google.com" target="_blank">Google</a>

<a href="tel:+254700123456">Call</a>

5.2 Images

‚ö†Ô∏è ALWAYS write alt text! It helps blind users and search engines.

Modern Tags for Structure

These tags describe WHAT the content is, not how it looks.

Tag Purpose

Chapter 9: Best Practices

‚úÖ Good Practices:

  1. Always close tags
  2. Use lowercase: <div> not <DIV>
  3. Always write alt text for images
  4. Use semantic tags
  5. Indent nested elements
  6. Add comments for clarity
  7. Only one <h1> per page

‚ùå Common Mistakes:

  • Forgetting closing tags
  • No alt text on images
  • Using uppercase tags
  • Multiple <h1> tags
  • Mixing <b> instead of <strong>

Common Attributes

Attribute Purpose
id Unique identifier (one per page)
class Group elements (can repeat)
style Inline CSS
title Tooltip text
