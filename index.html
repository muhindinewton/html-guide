Chapter 1: Introduction to HTML

üí° What is HTML?

HTML stands for HyperText Markup Language.

  • HyperText means text with links to other documents
  • Markup means we "mark up" text to give it structure
  • Language means it has rules and syntax

Important Things to Know:

  • ‚úÖ HTML is NOT a programming language‚Äîit's a markup language
  • ‚úÖ It tells your web browser what to display and how to organize content
  • ‚úÖ We are using HTML5, the latest version
  • ‚úÖ Every website you visit uses HTML!
üåç Real-World Example:

Think of HTML like the skeleton of a building:

  • The skeleton (HTML) gives structure
  • Paint and decorations (CSS) make it look beautiful
  • Electricity and plumbing (JavaScript) make it functional

Chapter 2: Your First Webpage Structure

üèóÔ∏è The Basic HTML Skeleton

Every HTML page MUST have this structure. Memorize it!

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Webpage</title>
</head>
<body>
    <h1>Hello World!</h1>
    <p>This is my first webpage.</p>
</body>
</html>

üìù Understanding Each Part:

Tag Purpose
<!DOCTYPE html> Tells the browser "This is HTML5"
<html lang="en"> Root element. lang="en" means English language
<head> Contains information ABOUT the page (not visible to users)
<meta charset="utf-8"> Supports special characters like √°, √±, ‚Ç¨, ‰∏≠
<meta name="viewport"...> Makes your website work well on mobile phones
<title> Shows in browser tab (very important!)
<body> Everything users SEE goes here
üí° Pro Tip: Always start EVERY HTML file with this skeleton. You can copy and paste it!

Chapter 3: Working with Text

3.1 Headings

HTML has 6 levels of headings, from biggest (<h1>) to smallest (<h6>).

<h1>Main Title (Use only ONCE per page)</h1>
<h2>Section Title</h2>
<h3>Sub-section</h3>
<h4>Smaller heading</h4>
<h5>Even smaller</h5>
<h6>Tiniest heading</h6>
‚ö†Ô∏è Important Rule: Use only ONE <h1> per page. It's like the title of a book‚Äîthere's only one main title!

3.2 Paragraphs

<p>This is a normal paragraph. Browsers automatically add space before and after it.</p>

3.3 Line Breaks and Horizontal Lines

This is line one<br>
This is line two

<hr>  <!-- Creates a horizontal line -->
üí° Note: <br> and <hr> are self-closing tags‚Äîthey don't need a closing tag!

3.4 Text Formatting Tags

Tag Purpose Example

Comments are notes in your code that browsers ignore. They help you remember what your code does!

üí° Use comments to:

  • Leave notes for yourself or teammates

Chapter 4: Creating Lists

4.1 Unordered Lists (Bullet Points)

Use when the order doesn't matter (like a shopping list).

<ul>
    <li>Milk</li>
    <li>Bread</li>
    <li>Butter</li>
</ul>
Result:
  • Milk
  • Bread
  • Butter

4.2 Ordered Lists (Numbers)

Use when the order matters (like recipe steps).

<ol>
    <li>Mix flour and water</li>
    <li>Add eggs</li>
    <li>Bake for 30 minutes</li>
</ol>
Result:
  1. Mix flour and water
  2. Add eggs
  3. Bake for 30 minutes

Customizing Ordered Lists:

<ol start="5" type="A">
    <li>Math</li>
    <li>English</li>
</ol>

Attributes:

  • start="5" ‚Üí starts counting from 5
  • type="A" ‚Üí uses letters (A, B, C...)
  • type="a" ‚Üí lowercase letters
  • type="I" ‚Üí Roman numerals (I, II, III...)
  • type="i" ‚Üí lowercase Roman numerals

4.3 Nested Lists

You can put lists inside lists!

<ul>
    <li>School Subjects
        <ol>
            <li>Mathematics</li>
            <li>Science</li>
            <li>English</li>
        </ol>
    </li>
    <li>Hobbies
        <ul>
            <li>Football</li>
            <li>Reading</li>
        </ul>
    </li>
</ul>

Chapter 5: Adding Links and Images

5.1 Links (Anchor Tag)

Links let users jump to other pages. They use the <a> tag (anchor).

<a href="https://www.google.com">Visit Google</a>

Important Attributes:

Attribute Purpose Example
href Where the link goes (required!) href="https://google.com"
target How to open the link target="_blank" (new tab)
title Tooltip when you hover title="Search Engine"
üìù Complete Example:
<a href="https://www.bbc.com/news" target="_blank" title="Latest News">
    Read BBC News
</a>

Special Types of Links:

<!-- Email link -->
<a href="mailto:teacher@school.com">Email Your Teacher</a>

<!-- Phone link -->
<a href="tel:+254700123456">Call Me</a>

<!-- Link to another page in your website -->
<a href="about.html">About Us</a>

5.2 Images

Images make websites interesting! Use the <img> tag.

<img src="puppy.jpg" alt="Cute brown puppy playing" width="400" height="300">

Essential Attributes:

Attribute Purpose Required?
src Image file location ‚úÖ YES
alt Description (for blind users and if image fails) ‚úÖ YES
width Width in pixels Optional
height Height in pixels Optional
‚ö†Ô∏è ALWAYS write alt text! It helps:
  • Blind users understand your images
  • Search engines find your website
  • Users know what image was supposed to be there if it fails to load
üìù Good vs Bad Examples:
<!-- ‚ùå BAD: No alt text -->
<img src="cat.jpg">

<!-- ‚úÖ GOOD: Has descriptive alt text -->
<img src="cat.jpg" alt="Orange tabby cat sleeping on sofa">

Using Online Images:

<img src="https://placehold.co/600x400" alt="Placeholder image">

Chapter 6: Building Tables

Tables organize data in rows and columns (like Excel spreadsheets).

6.1 Basic Table Structure

<table border="1">
    <tr>
        <th>Student</th>
        <th>Math</th>
        <th>English</th>
    </tr>
    <tr>
        <td>Ali</td>
        <td>92</td>
        <td>88</td>
    </tr>
    <tr>
        <td>Amina</td>
        <td>85</td>
        <td>90</td>
    </tr>
</table>

Understanding Table Tags:

Tag Meaning
<table> Container for entire table
<tr> Table Row
<th> Table Header (bold + centered)
<td> Table Data (normal cell)

6.2 Professional Table with All Parts

<table border="1" cellpadding="10" cellspacing="0">
    <caption>Grade 8A Term 1 Results</caption>
    
    <thead>
        <tr>
            <th>Student Name</th>
            <th>Math</th>
            <th>Science</th>
        </tr>
    </thead>
    
    <tbody>
        <tr>
            <td>John</td>
            <td>85</td>
            <td>90</td>
        </tr>
        <tr>
            <td>Mary</td>
            <td>92</td>
            <td>88</td>
        </tr>
    </tbody>
</table>

Extra Tags Explained:

  • <caption> - Table title (appears above table)
  • <thead> - Groups header rows
  • <tbody> - Groups body rows
  • cellpadding - Space inside cells
  • cellspacing - Space between cells

Chapter 7: Creating Forms

Forms collect information from users (like registration, login, surveys).

7.1 Basic Form Structure

<form action="/submit" method="POST">
    <!-- Form inputs go here -->
    <input type="submit" value="Send">
</form>

Form Attributes:

  • action - Where to send the data
  • method - How to send it (POST or GET)

7.2 Text Inputs

<form>
    <label for="username">Username:</label><br>
    <input type="text" id="username" name="username" placeholder="Enter your name" required>
    <br><br>

    <br><br>
    
    <label for="pass">Password:</label><br>
    <input type="password" id="pass" name="password" minlength="8">
</form>
üí° Always pair labels with inputs! The for attribute in <label> must match the id in <input>.

Chapter 8: Modern HTML5 Elements

üéØ What are Semantic Tags?

Semantic means "meaningful." These tags describe what the content IS, not how it looks.

8.1 Why Use Semantic Tags?

  • ‚úÖ Search engines (Google) understand your content better
  • ‚úÖ Screen readers help blind people navigate
  • ‚úÖ Your code is easier to read and organize
  • ‚úÖ Better for teamwork

8.2 Common Semantic Tags

Tag Purpose Example Use
<header> Top section of page Logo, navigation menu
<nav> Navigation links Menu bar
<main> Main content (only ONE per page) The actual page content
<article> Self-contained content Blog post, news article
<section> Thematic group of content Chapters in a page
<aside> Side content Sidebar, advertisements
<footer> Bottom of page Copyright, contact info

8.3 Complete Page Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <title>My Blog</title>
</head>
<body>
    
    <header>
        <h1>Welcome to My Blog</h1>
        <nav>
            <a href="#home">Home</a> |
            <a href="#about">About</a> |
            <a href="#contact">Contact</a>
        </nav>
    </header>
    
    <main>
        <article>
            <h2>My First Blog Post</h2>
            <p>This is the content of my article...</p>
        </article>
        
        <section>
            <h2>Comments</h2>
            <p>Great article! - John</p>
        </section>
    </main>
    
    <aside>
        <h3>About Me</h3>
        <p>I'm a Grade 8 student who loves coding!</p>
    </aside>
    
    <footer>
        <p>&copy; 2024 My Blog. All rights reserved.</p>
    </footer>
    
</body>
</html>

8.4 Figure and Figcaption

Use these to add captions to images:

<figure>
    <img src="sunset.jpg" alt="Beautiful sunset over ocean">
    <figcaption>Sunset at Diani Beach, Kenya</figcaption>
</figure>

8.5 Div vs Semantic Tags

‚ùå OLD WAY (avoid this): 
<div class="header">...</div>
<div class="nav">...</div>
<div class="footer">...</div>
‚úÖ NEW WAY (use this): 
<header>...</header>
<nav>...</nav>
<footer>...</footer>

8.6 When to Use Div and Span

  • <div> - Block container (takes full width, new line)
  • <span> - Inline container (stays in line)

Rule: Only use <div> and <span> when no semantic tag fits!

<p>The word <span style="color:red;">red</span> is colored.</p>

<div class="warning-box">
    <p>This is a special container.</p>
</div>

Chapter 9: Best Practices & Common Mistakes

9.1 Common HTML Attributes

Attribute Purpose Usage
id Unique identifier (only ONE per page) <div id="header">
class Group elements (can repeat) <p class="important">
style Inline CSS (avoid if possible) <p style="color:blue;">
title Tooltip on hover <a title="Click here">

9.2 HTML Rules (Do These!)

‚úÖ Good Practices:

  1. Always close your tags properly
  2. Use lowercase for tag names: <div> not <DIV>
  3. Indent nested elements for readability
  4. Always write alt text for images
  5. Use semantic HTML5 tags
  6. Include <!DOCTYPE html> at the top
  7. Add comments to explain complex code
  8. Validate your HTML (use W3C Validator)

9.3 Common Mistakes (Avoid These!)

‚ùå Common Errors:

1. Forgetting closing tags:

<!-- ‚ùå Wrong -->
<p>Hello
<p>World

<!-- ‚úÖ Correct -->
<p>Hello</p>
<p>World</p>

2. Mixing up <strong> and <b>:

<!-- ‚ùå Bad (just visual) -->
<b>Important!</b>

<!-- ‚úÖ Good (meaningful) -->
<strong>Important!</strong>

3. No alt text on images:

<!-- ‚ùå Bad -->
<img src="photo.jpg">

<!-- ‚úÖ Good -->
<img src="photo.jpg" alt="School graduation ceremony">

4. Using uppercase tags:

<!-- ‚ùå Wrong -->
<DIV><P>Text</P></DIV>

<!-- ‚úÖ Correct -->
<div><p>Text</p></div>

5. Multiple <h1> tags:

<!-- ‚ùå Bad -->
<h1>Welcome</h1>
<h1>About Us</h1>

<!-- ‚úÖ Good -->
<h1>Welcome</h1>
<h2>About Us</h2>

9.4 Quick Reference Checklist

Before finishing your HTML page, check these:

  • ‚òê <!DOCTYPE html> at the top?
  • ‚òê All tags properly closed?
  • ‚òê All images have alt text?
  • ‚òê Only one <h1> per page?
  • ‚òê Used semantic tags instead of divs?
  • ‚òê Proper indentation (readable)?
  • ‚òê No spelling errors in content?
  • ‚òê All links work?
